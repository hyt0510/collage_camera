# Collage Camera システム構成図

## 1. 概要
本システムは、文化祭等のイベントで来場者が複数の写真を撮影し、一つのコラージュ作品として共用モニターに投稿するためのWebアプリケーションである。
「1人1回」の投稿制限を持ちつつ、運営側が「プリセット（枠組みやテーマのセット）」を変更した際には、同じユーザーが再度新しい作品を投稿できる仕組みを備えている。

## 2. アーキテクチャ
- **Frontend**: Next.js (App Router), TypeScript, Tailwind CSS
- **Backend**: Next.js Route Handlers
- **Database**: Firebase Firestore (投稿データ管理)
- **Storage**: Firebase Storage (画像ファイル管理)

## 3. 主要コンポーネントと責務

### クライアントサイド
- **`app/page.tsx`**: メイン画面。投稿状態（未投稿/投稿済み/完了）に応じた UI 制御を行う。
- **`hooks/useCollageCapture.ts`**: 
    - サーバーから最新のプリセットを取得。
    - ローカルストレージ (`collage_v10_submitted`) に保存された「最後に投稿したプリセットID」と現在のIDを比較し、再投稿の可否を判定。
    - 写真の最適化（リサイズ・圧縮）およびコラージュ画像の生成。
- **`components/features/collage/CollageFrame.tsx`**: プリセットに基づいた撮影インターフェースを提供。

### サーバーサイド
- **`lib/submission-store.ts`**: 
    - 投稿データのバリデーションと Firestore への保存。
    - **重複チェック論理**: `userId` と `presetId` の組み合わせで既存データを検索し、同一プリセット内での多重投稿を防止。
- **`lib/capture-preset-store.ts`**: 現在アクティブなプリセット（枠組み、テーマ、ID）の管理。
- **`app/api/submissions/route.ts`**: 投稿受付 API。検閲機能（Cloud Vision API 等）との連携も可能。

## 4. データ構造 (Submission)
Firestore の `submissions` コレクションに以下の形式で保存される。

| フィールド | 型 | 説明 |
| :--- | :--- | :--- |
| `id` | string | 投稿ID |
| `userId` | string | ユーザー識別子（ブラウザ生成のID） |
| `presetId` | string | 投稿時のプリセットID（再投稿判定に使用） |
| `templateId` | string | 使用された枠組みテンプレートID |
| `status` | string | 公開ステータス (approved / pending / hidden) |
| `placement` | object | モニター上の配置位置 (row, col, label) |
| `items` | array | 各枠の写真URLとテーマのリスト |
| `collageImageUrl` | string | 生成されたコラージュ画像全体のStorage URL |

## 5. 再投稿可能化のロジック
1. 運営が管理画面等でプリセット（`presetId`）を更新する。
2. ユーザーのブラウザがリロードまたはアクセスした際、最新の `presetId` を取得。
3. ローカルストレージに保存されている `submitted_preset_id` と現在の `presetId` が一致しない場合、投稿制限を解除する。
4. 投稿成功時、新しい `presetId` をローカルストレージに保存する。
5. サーバー側でも `userId` + `presetId` のペアでチェックを行うため、不正な連投を確実に防止する。
